Contributing to Multiflex Core
==============================

Thanks for your interest in contributing!

This project is a fork/derivative of Bitcoin Core and therefore shares a similar development model: contributions are welcome in the form of peer review, testing, documentation improvements, and patches.

How to contribute
-----------------

- **Search existing issues** before opening a new one.
- For changes, please open a **Pull Request**.
- Keep changes focused and easy to review.
- If your change affects consensus, networking, wallets, or security-sensitive code, include tests and clear reasoning.

Coding style / tooling
----------------------

The repository contains configuration and tooling in `doc/` and `contrib/`.
Please follow the existing code style in the area you are touching.

Developer Certificate of Origin
-------------------------------

If your organization requires a DCO sign-off, you may add `Signed-off-by:` lines to your commits.
(If you don't use DCO, you can remove this section.)
